# Changelog

## v1.109.0 — arianee-sdk — Improved event deep link support and smart asset merging, plus security and compatibility fixes

This release adds deep link domain support to event content, fixes asset merging to preserve parent and child external content, and addresses token recovery timestamp logic. Dependency updates include a lodash security fix and revert of incompatible lint/build upgrades.

> **Events now expose a deeplinkDomain property, enabling dynamic deep link resolution.**

## 🔗 Event and smart asset handling

- Event content now includes a deeplinkDomain property for flexible smart link resolution.
- Child externalContents are now appended to parents in asset merges, preventing data loss.
- Token recovery timestamp calculation updated to allow recovery flows for NFTs minted over 5 years ago.

## 🛠️ Smart link and redirection fixes

- createLink now accepts identity address as a parameter for redirection and OA flows.
- Fixed createLink to handle both query params and path params for product page redirection.

## 🧑‍💻 Dependency and compatibility updates

- Upgraded @arianee/arianee-abi to 0.37.2 to address lodash CVE-2026-4800 (CVSS 8.1).
- Reverted incompatible lint/build dependency upgrades to restore CI compatibility with Angular 16/NX 16.
- Other minor dependency updates: axios, express, qs, react-router, esbuild, hono, tar, diff, min-document.
